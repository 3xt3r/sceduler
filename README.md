Attack Vector — AV:N
Вопрос

Какой вектор атаки используется для эксплуатации уязвимости?

Страница /customers доступна через сайт компании и может быть открыта удалённо через браузер.

Варианты ответа
Сетевой — AV:N / Network

Правильный ответ.

Уязвимость может быть эксплуатирована удалённо через сеть, например через интернет. Атакующему достаточно открыть доступный URL /customers без авторизации.

Смежная сеть — AV:A / Adjacent

Не подходит.

Этот вариант используется, если атакующий должен находиться в той же локальной сети, Wi-Fi, VPN-сегменте или другом смежном сетевом окружении.

Локальный — AV:L / Local

Не подходит.

Локальный вектор используется, если атакующему нужен доступ к локальной системе, терминалу, пользовательской сессии или локальному приложению.

Физический — AV:P / Physical

Не подходит.

Физический доступ к устройству или серверу для эксплуатации этой уязвимости не требуется.

2. Attack Complexity — AC:L
Вопрос

Какая сложность эксплуатации уязвимости?

Для просмотра данных достаточно открыть страницу /customers, которая ошибочно доступна без авторизации.

Варианты ответа
Низкая — AC:L / Low

Правильный ответ.

Эксплуатация не требует специальных условий, обхода защитных механизмов, гонки состояний или сложной подготовки. Достаточно отправить обычный HTTP-запрос или открыть страницу в браузере.

Высокая — AC:H / High

Не подходит.

Высокая сложность выбирается, если для эксплуатации нужны особые условия, редкая конфигурация, точное время выполнения, сложная последовательность действий или нестабильный сценарий эксплуатации.

3. Privileges Required — PR:N
Вопрос

Какие привилегии нужны атакующему для эксплуатации уязвимости?

Страница /customers доступна всем пользователям без авторизации.

Варианты ответа
Не требуются — PR:N / None

Правильный ответ.

Атакующему не нужна учётная запись, роль пользователя, админ-доступ или какие-либо права в системе. Уязвимость доступна анонимно.

Низкие — PR:L / Low

Не подходит.

Этот вариант выбирается, если атакующему нужна обычная пользовательская учётная запись, но не административные права.

Высокие — PR:H / High

Не подходит.

Этот вариант используется, если для эксплуатации нужны привилегии администратора или другого высокоуровневого пользователя.

4. User Interaction — UI:N
Вопрос

Требуется ли взаимодействие другого пользователя для эксплуатации уязвимости?

Атакующий может самостоятельно открыть страницу /customers.

Варианты ответа
Не требуется — UI:N / None

Правильный ответ.

Для эксплуатации не нужно, чтобы жертва переходила по ссылке, открывала файл, подтверждала действие или выполняла какие-либо операции. Атакующий действует самостоятельно.

Требуется — UI:R / Required

Не подходит.

Этот вариант используется, если эксплуатация возможна только при участии другого пользователя, например если жертва должна открыть ссылку, загрузить файл или нажать кнопку.

5. Scope — S:U
Вопрос

Меняется ли область воздействия уязвимости?

Уязвимость находится в веб-приложении компании и раскрывает данные, которыми управляет это же приложение.

Варианты ответа
Не изменяется — S:U / Unchanged

Правильный ответ.

Эксплуатация влияет на тот же компонент безопасности, в котором находится уязвимость. Ошибка авторизации в веб-приложении приводит к раскрытию данных этого же веб-приложения.

Изменяется — S:C / Changed

Не подходит.

Этот вариант выбирается, если уязвимость в одном компоненте позволяет повлиять на другой компонент с отдельной областью безопасности. Например, ошибка в одном сервисе позволяет получить доступ к другой системе, другому приложению или инфраструктурному компоненту.

6. Confidentiality Impact — C:H
Вопрос

Оказывает ли эксплуатация уязвимости влияние на конфиденциальность?

Страница /customers раскрывает список клиентов: ФИО, email и телефон.

Варианты ответа
Не оказывает — C:N / None

Не подходит.

Этот вариант означал бы, что эксплуатация не раскрывает никаких защищаемых данных. В данном случае данные клиентов доступны неавторизованному пользователю, поэтому влияние на конфиденциальность есть.

Низкое — C:L / Low

Может подойти только в более ограниченном случае.

Например, если раскрывается небольшое количество тестовых записей или ограниченный объём не особо чувствительной информации. Но если страница показывает реальную клиентскую базу или значительную её часть, этого уровня недостаточно.

Высокое — C:H / High

Правильный ответ для указанного вектора.

Уязвимость позволяет неавторизованному пользователю получить доступ к клиентским данным: ФИО, email и телефону. Если раскрывается весь список клиентов или значительная часть клиентской базы, это считается серьёзным нарушением конфиденциальности.

7. Integrity Impact — I:N
Вопрос

Оказывает ли эксплуатация уязвимости влияние на целостность данных?

На странице /customers можно только просматривать данные, но нельзя их изменять или удалять.

Варианты ответа
Не оказывает — I:N / None

Правильный ответ.

Атакующий не может изменять, удалять, создавать или подменять данные клиентов. Следовательно, влияние на целостность отсутствует.

Низкое — I:L / Low

Не подходит.

Этот вариант был бы уместен, если бы атакующий мог ограниченно изменять часть данных, например исправить телефон или email отдельного клиента.

Высокое — I:H / High

Не подходит.

Высокое влияние на целостность было бы, если бы атакующий мог массово изменять данные, удалять клиентов, подменять важную информацию или нарушать корректность работы системы.

8. Availability Impact — A:N
Вопрос

Оказывает ли эксплуатация уязвимости влияние на доступность сервиса или данных?

Открытие страницы /customers не приводит к отказу в обслуживании и не нарушает работу приложения.

Варианты ответа
Не оказывает — A:N / None

Правильный ответ.

Эксплуатация уязвимости не делает сервис недоступным, не блокирует пользователей и не нарушает работу приложения.

Низкое — A:L / Low

Не подходит.

Этот вариант выбирается, если эксплуатация частично снижает доступность, например вызывает временные задержки, частичные сбои или ограниченное ухудшение работы.

Высокое — A:H / High

Не подходит.

Высокое влияние на доступность было бы, если бы эксплуатация могла полностью вывести сервис из строя, сделать данные недоступными или вызвать серьёзный отказ в обслуживании.
